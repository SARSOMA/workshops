# Section 1: Introduction & Setup (Foundation)

## Overview

This section introduces GitHub Copilot CLI and ensures all participants have a working environment. By the end, everyone will understand what Copilot CLI is, its role in AI-driven development, and have a fully functional setup.

## Learning Objectives

By the end of this section, you will be able to:
- Explain what GitHub Copilot CLI is and its role in the developer workflow
- Identify the key differences between Copilot CLI and other Copilot experiences
- Successfully launch and authenticate GitHub Copilot CLI
- Navigate the basic interface and understand the command structure
- Understand the security model and trust system

## Prerequisites

- Completed pre-workshop setup (see [Prerequisites](../00-Prerequisites/README.md))
- GitHub account with Copilot access
- Terminal access (PowerShell 6+ on Windows, any terminal on macOS/Linux)

## Key Takeaways

- GitHub Copilot CLI is a terminal-native AI assistant powered by the same agentic harness as GitHub's Copilot coding agent
- It provides full GitHub integration out of the box
- Security is built-in with explicit approval for file modifications and command execution
- Custom instructions and MCP servers enable extensibility
