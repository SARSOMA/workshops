# Section 8: End-to-End Developer Workflows (Hands-on Scenarios)

## Overview

This section provides practical, hands-on scenarios that combine everything you've learned. Each scenario represents a real-world developer workflow that you can practice and adapt to your work.

## Learning Objectives

By the end of this section, you will be able to:
- Create and manage pull requests entirely from CLI
- Review code changes effectively using Copilot
- Address PR feedback efficiently
- Debug pipeline failures with Copilot assistance
- Troubleshoot issues systematically

## Scenarios in This Section

| Scenario | Duration | Description |
|----------|----------|-------------|
| [8.1 PR Workflow](./01-pr-workflow.md) | 20 min | Complete PR lifecycle |
| [8.2 Code Review](./02-code-review.md) | 15 min | Reviewing changes |
| [8.3 Addressing Feedback](./03-addressing-feedback.md) | 15 min | Handling PR comments |
| [8.4 Pipeline Debugging](./04-pipeline-debugging.md) | 20 min | Fixing CI/CD failures |

## Prerequisites

- Completed Sections 1-6
- A repository to practice with
- (Optional) ADO/GitHub pipeline access for pipeline scenario
