# Section 5: End-to-End Workflows

## Overview

This is the **hands-on project section** where you'll apply everything you've learned using a real open-source project: **[SARSOMA/akri](https://github.com/SARSOMA/akri)**.

You'll practice:
- Understanding a new codebase
- Making changes via Copilot CLI
- Creating pull requests
- Reviewing code and adding comments

## Learning Objectives

By the end of this section, you will be able to:
- Quickly understand an unfamiliar codebase using Copilot CLI
- Make targeted code changes via natural language prompts
- Create and manage pull requests from the command line
- Review code and leave comments using Copilot CLI

## Duration

**35 minutes**

## Prerequisites

- Completed Sections 1-4
- GitHub account
- Git configured locally
- (Optional) Fork of SARSOMA/akri for creating PRs

## Project: SARSOMA/akri

**Akri** is a Kubernetes resource interface for the edge. It's a good workshop project because:
- Public repository (anyone can view)
- Well-structured Rust codebase
- Active development with issues and PRs
- Good documentation

## Key Takeaways

- Copilot CLI excels at exploring unfamiliar codebases
- Natural language makes complex Git workflows accessible
- Code review from CLI is surprisingly effective
- End-to-end workflows without leaving your terminal
