# Part 3: Enterprise Agents (Microsoft 365 Copilot)

Parts 1 and 2 covered developer-facing agents — markdown files that steer the model and programmatic extensions that run as web services. This part covers the **enterprise side**: agents built inside the Microsoft 365 Copilot platform, designed for business users and distributed through the org.

These agents live in the M365 ecosystem. They appear inside Microsoft 365.

You can take a look inside [Copilot Studio](https://copilotstudio.preview.microsoft.com/environments/0e70b315-2fc8-e65e-8c0f-cc9894abbf10/home) to look into what built in integration there is.

I find this best for aggregating data over user facing apps (teams, outlook) in a no-code fashion. For example, a pipeline failure teams notification bot.
